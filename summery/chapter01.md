# Chapter 01 金融机器学习为何是独立学科

## 本章一句话
金融机器学习的难点不在“模型复杂度”，而在“数据生成机制、验证方法、研究流程”与传统机器学习完全不同。

## 核心内容
1. 金融数据通常非独立同分布（non-IID），存在自相关、异方差、结构突变、反馈效应。
2. 同一个策略在样本内有效，样本外可能立即失效，说明泛化目标比拟合目标重要得多。
3. 项目失败常见原因不是算法落后，而是实验设计错误、泄漏、错误指标和错误回测。
4. 研究必须把统计显著性、经济显著性、可执行性三者统一。

## 方法论框架（全书地图）
1. 数据结构：先解决采样偏差和信息表达（Ch2）。
2. 标签定义：把“未来结果”定义成可交易问题（Ch3）。
3. 样本权重：处理重叠与不独立（Ch4）。
4. 特征处理：平稳性与记忆保留平衡（Ch5）。
5. 建模与验证：集成学习+防泄漏交叉验证（Ch6-9）。
6. 下注与组合：把预测转化为仓位和资产配置（Ch10, Ch16）。
7. 回测与风控：防过拟合、评估策略失效概率（Ch11-15）。

## 本章给出的研究标准
1. 任何结果都必须有样本外证据。
2. 验证切分必须匹配交易时间结构。
3. 模型输出必须可映射到仓位、风险预算和执行约束。
4. 研究流程要可复现、可审计、可扩展。

## 常见误区
1. 直接用 Kaggle 风格特征工程和 K-Fold。
2. 先调模型，后想标签与采样。
3. 只看准确率，不看收益路径和交易成本。
4. 把一次漂亮回测当成策略真实性证明。

## 实务落地清单
1. 在立项阶段先定义验证协议，再写模型代码。
2. 明确“策略假设失效条件”和重训触发规则。
3. 把实验日志、参数、切分方案固化到版本管理。
