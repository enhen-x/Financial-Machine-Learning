# Chapter 16 Machine Learning Asset Allocation

## 本章一句话
传统均值-方差优化对参数误差极端敏感，层次化风险分配（HRP）是更稳健的替代。

## 核心内容
1. 凸优化框架在估计误差下会产生不稳定权重（Markowitz curse）。
2. 协方差矩阵微小扰动可导致大幅换仓和集中持仓。
3. HRP 通过相关性聚类和层次分配，把“几何距离”转为“层次结构”来配置权重。
4. 数值与 Monte Carlo 样本外实验显示，HRP 通常在稳健性上优于经典方案。

## HRP 核心流程
1. 估计相关矩阵并转成距离矩阵。
2. 做层次聚类，得到资产树结构。
3. 进行准对角化排序。
4. 递归二分分配风险预算，生成组合权重。

## 实战价值
1. 降低对预期收益估计的依赖。
2. 提升权重稳定性与可解释性。
3. 在高维资产、低样本场景更实用。

## 常见误区
1. 把 HRP 当成“无参数万能法”，忽略相关估计质量。
2. 忽略交易成本与调仓频率约束。
3. 不做样本外比较就替换原有配置框架。

## 实务落地清单
1. 与最小方差、风险平价做长期并行对照。
2. 设置换手上限，防止相关矩阵噪声引发过度调仓。
3. 在危机与平稳样本分层评估 HRP 适用性。
