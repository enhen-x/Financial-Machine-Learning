# 数据处理执行规范（基于《Advances in Financial Machine Learning》）

本目录下的所有数据处理脚本，统一遵循以下规范。除非明确说明并记录偏离原因，否则不得跳过。

## 1. 采样与数据组织

1. 优先评估信息驱动采样（Dollar/Volume/Tick Bars），不要默认使用固定时间采样。
2. 若使用时间采样，需在脚本注释中说明原因，并评估噪声与异方差影响。

## 2. 可交易口径清洗

1. 必须处理停牌、涨跌停、一字板、退市、复权、名称变更/ST。
2. 训练样本必须满足“当时可交易、可执行”的时点口径。

## 3. 时间可得性与防前视

1. 所有特征仅允许使用标签时点可获得信息。
2. 严禁使用未来数据构造当前特征（Look-ahead Bias）。

## 4. 标签工程

1. 优先使用事件驱动标签（如 Triple-Barrier），不默认使用“未来 N 日涨跌”粗标签。
2. 标签参数（持有期、止盈、止损）必须显式记录。

## 5. 样本重叠与信息泄露控制

1. 必须评估事件窗口重叠（concurrency）。
2. 对重叠样本使用唯一性相关处理（如 uniqueness weighting）降低信息重复偏置。

## 6. 非平稳性处理

1. 特征应优先平稳化处理。
2. 需要时使用分数差分（Fractional Differentiation）以平衡记忆保留与平稳性。

## 7. 训练/验证切分规范

1. 禁止随机切分时间序列样本。
2. 默认使用 Purged K-Fold + Embargo，防止标签窗口泄露。

## 8. 类别不平衡处理

1. 对标签分布失衡进行显式处理（阈值、重采样、代价敏感学习等）。
2. 需记录处理前后标签分布。

## 9. 样本加权与评估口径

1. 样本默认不等权，优先按信息独特性/并发度加权。
2. 评估指标需与策略目标一致，并区分样本内与样本外表现。

## 10. 可复现与审计

1. 每次处理必须记录：数据版本、过滤规则、标签参数、特征参数、切分区间。
2. 任何偏离本规范的实现，必须在脚本与文档中注明原因和风险。

## 强制执行清单（每次处理前后都要勾选）

- [ ] 可交易口径已检查（停牌/涨跌停/ST/复权/退市）
- [ ] 特征无前视偏差
- [ ] 标签为事件驱动或已说明替代原因
- [ ] 已处理样本重叠与唯一性
- [ ] 使用 Purged + Embargo 切分
- [ ] 已处理类别不平衡
- [ ] 已记录全部参数与数据版本
